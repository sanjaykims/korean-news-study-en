3 Daily Ingestion Flow

  1. 1
    Cron triggers at 01:00 KST → calls /api/auto-ingest
  2. 2
    Searches YouTube for yesterday's JTBC 뉴스룸 다시보기 — strict filter requires year-prefixed date in title (e.g. 26.4.14, 2026.04.14) to prevent cross-year false matches with old videos
  3. 3
    Calls Cloudflare Worker with the video ID
  4. 4
    Worker fetches metadata (title, chapters) via YouTube Data API
  5. 5
    Worker fetches transcript via Supadata API
  6. 6
    Extracts actual broadcast date from title pattern (YY.M.DD) — JTBC often uploads replays days late
  7. 7
    Splits into articles using YouTube chapters as boundaries
  8. 8
    Claude classifies topics (정치/경제/사회/국제 etc.)
  9. 9
    Saves to Supabase — articles appear on your calendar under the correct date

4 Study Flow

  1. A
    VideoStep — watch JTBC clip with timestamp
  2. B
    ScriptStep — read transcript, tap words for hanja + Chinese. Level switcher (原文 / 初级 / 中级 / 高级) lets you read the same news at TOPIK 1–6 difficulty. Rewrites are Claude-generated, news-register preserved (date markers, quote attribution, numbers, titles all retained), and cached in DB per article.
  3. C
    QuizStep — bidirectional Chinese↔Korean translation drill
  4. D
    ShadowingStep — record yourself, get pronunciation score. 结束练习 button lets you end early at any point — partial results are saved and vocabulary is persisted.
  5. R
    报告 PDF button (study page header) — generates a downloadable A4 PDF: header (date · title · topic · reporter), full proofread script, selected vocabulary table with hanja + Chinese, and the local generation timestamp. Every download fires a report_generated event (user · article · timestamp · word count) into learning_events.

5 Multi-Level Rewrites

News-register difficulty levels
On ScriptStep, tap 原文 / 初级 / 中级 / 高级 to read the same JTBC news at 4 difficulty levels.

Generation: on-demand via POST /api/rewrite → Claude Sonnet 4.5. First view costs ~10 s and ~$0.01; cached forever in news_articles.transcript_proofread as JSON {beginner, intermediate, advanced}. Re-opens are instant.

News authenticity: the prompt mandates preserving date markers, numbers, place names, quote attributions ("…"라고 말했습니다), and 선수/기자 titles at every level. Only vocabulary and grammatical density change.

Level targets: beginner ≈ TOPIK 1–2 (단문, 기본 문법), intermediate ≈ TOPIK 3–4 (연결어미, 적당한 한자어), advanced ≈ TOPIK 5–6 (보도 문체, 복합 문법). Original = unmodified anchor transcript.

Schema-free: rewrites are JSON-encoded into the existing transcript_proofread TEXT column. No migration required.

6 SRS Review System

Spaced Repetition (간격 반복)
Words saved during ScriptStep are tracked in vocabulary_log with a mastery_level (0–5).

Interval schedule: level 0 = immediate, 1 = 1 day, 2 = 3 days, 3 = 7 days, 4 = 14 days, 5 = 30 days (mastered).

Correct → mastery +1 (longer interval). Incorrect → mastery −1 (shorter interval).

Homepage → SRS复习 button → flashcard quiz of up to 15 due words → results update mastery_level + last_reviewed_at in DB.

Each card shows: Chinese/meaning prompt, hanja hint, current mastery badge, review count, word origin. After answer reveal, both buttons preview the next mastery level and SRS interval.

10 Failure Recovery

If content stops appearing:
1. Check /api/articles — if it returns fetch failed, Supabase paused. Unpause from dashboard.

2. If error says 다시보기를 찾지 못했습니다: JTBC hasn't uploaded the replay yet (2-4 day lag is normal). Wait a couple days and retry — never override with a manual videoId unless you're sure it's the right broadcast.

3. If errors mention SUPADATA or YT_DATA_API: an API key expired. Generate a new one and update Cloudflare Worker secret.

4. If a date is in Supabase but missing from the calendar: the /api/dates route must use force-dynamic + admin client. Already fixed — never let anyone "optimize" it back to cached.

5. If manual ingest picks the wrong (old) video: titleMatchesDate() in yt-proxy uses year-prefixed variants only (e.g. 26.4.14, 2026년 4월 14일). Year-less formats like 4.14 were removed because they caused cross-year false positives. Never re-add year-less date variants.

11 Dev Automation

Claude can now ship end-to-end
A GitHub Personal Access Token is stored in Claude's session memory, scoped to this repo with contents + pull-requests write. Claude can push to a claude/* branch, open a PR, and merge it via API in one go — no manual GitHub step needed.

Token expires per GitHub's max policy (1 year). If Claude reports "Bad credentials", regenerate at github.com/settings/personal-access-tokens.
